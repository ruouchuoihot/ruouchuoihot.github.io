# The Card

## Bối cảnh (Scenario)

Holmes vô tình nhặt được sơ hở từ cục bánh mì của Dr. Nicole Vale - toàn là mấy mớ tàn tích ráp lại sau vụ càng quét vỡ nát hệ thống Cogwork-1. Điểm thú vị là tất cả đường kim mũi chỉ đều chĩa mũi dui vào một tấm namecard số má ký tên: "JM".

> Case này đậm chất bóc bánh SOC và vọc CTI (Cyber Threat Intelligence). Lab thảy ra 3 cục log và một mâm tảng các platform CTI xịn xò để hành nghề.

![Platform](/assets/images/htb/the-card/fc171b0c-eb98-425d-b4c9-4d1c299b888a.png)

---

## Phân tích Chuyên sâu (Walkthrough)

### 1. Phân tích cú chạm ngõ đầu tiên (First User-Agent)

> Analyze the provided logs and identify what is the first User-Agent used by the attacker against Nicole Vale's honeypot.

Hàng lỏ Challenge cung cấp 3 bộ log là `waf.log`, `application.log` và `access.log`. Để đâm thủng được thông tin `first User-Agent` (User-Agent gõ cửa sớm nhất), mình lót tay vào bóc ruột con `access.log` lượn theo sườn thời gian.

![First User Agent Log](/assets/images/htb/the-card/5c72bf85-f114-4d5d-9666-bf6bfefe06eb.png)

Vớt lên màn hình một cái chuỗi quen thuộc:

```
Lilnunc/4A4D - SpecterEye
```

### 2. Phát hiện mầm độc Web Shell

> It appears the threat actor deployed a web shell after bypassing the WAF. What is the file name?

Lội tới bước này, để vạch trần con web shell được rặn ra lót ổ sau khi đục lớp WAF ngon lành, tôi móc mắt sang dòm log ở `waf.log`.
Soi sấp ngửa thì thấy vọt ra 1 event nhạy cảm: file `.php` mọc chình ình với quả action cắm biển là "bypass".

![WAF Bypass Logs](/assets/images/htb/the-card/e35afcfa-1687-42a7-9f9a-1adcd5280ccd.png)

File ngậm độc đó chính là:

```
temp_4A4D.php
```

### 3. Điều tra mẻ data bị cướp (Exfiltrated Data)

> The threat actor also managed to exfiltrate some data. What is the name of the database that was exfiltrated?

Ở cùng mớ log WAF trên, nếu tỉnh táo ngâm kỹ chút là chộp dính cái tên tệp Database lão attacker vừa bốc đi.

![Data Exfiltration](/assets/images/htb/the-card/3beef3ce-c134-495a-926c-abc5ec1827aa.png)

Tên file DB bị "hốt":

```
database_dump_4A4D.sql
```

### 4. Tìm manh mối đoạn chuỗi lặp (Recurring String)

> During the attack, a seemingly meaningless string seems to be recurring. Which one is it?

Bới trong cặn log `application.log`, một chuỗi gờn gợn cứ hiện hồn lặp đi chặp lại như phá bĩnh. Cụ thể là dãy hex:

![Application Logs String](/assets/images/htb/the-card/c7c31ada-9062-4e06-88c2-45441a218986.png)

```
4A4D
```

### 5. Khảo sát CTI Platform - Đếm số lượng Campaigns

> OmniYard-3 (formerly Scotland Yard) has granted you access to its CTI platform. Browse to the first IP:port address and count how many campaigns appear to be linked to the honeypot attack.

Truy cập thẳng tay vào platform CTI mà đề bài giao mâm. Filter móc theo họng "campaigns", ta quét vớt được tổng cộng 8 mảng chiến dịch (campaigns).

![CTI Campaigns](/assets/images/htb/the-card/5d7b44ce-4d87-4c1a-bf0d-16527b446a95.png)

Đãi nhặt từng mảnh Description (Mô tả) của đám campaigns này, văng ra một ròng chữ chứa cái signature khét lẹt "JM signature" (Mã hex là `4A4D`) chặp y đúc chuỗi đã soi được ở Câu 4. Đếm tới đếm lui thì thuổng trọn tổng cộng 5 campaigns vướng bẩn.

![CTI Signatures](/assets/images/htb/the-card/efd7485c-274d-49c9-b202-97aa676bc4ce.png)

```
5
```

### 6. Số đếm bộ Tools và Malware

> How many tools and malware in total are linked to the previously identified campaigns?

Từ chiến lợi phẩm ở câu 5, khoanh được 5 ổ campaigns sau:
- Operation Neural Storm
- Quantum Heist
- Civic Disruption
- Bio-Breach
- Transport Chaos

Sau đó, búp trảo vào mục **Links** của các gã này, ốp quả chặn Filter ngay thẻ Category (Malware và Tool) để soi list:

![CTI Links](/assets/images/htb/the-card/b1130c00-d2a3-4f35-aec8-d2395db21a48.png)

Chi tiết hàng họ nhét trong từng campaign:
- **Operation Neural Storm**: Malware `NeuroStorm Implant`, Tool `NeuroScan Pro`
- **Quantum Heist**: Malware `QuantumCoin Stealer`, Tool `QuantumKeyX`
- **Civic Disruption**: Malware `CityWide Disruptor`, Tool `CityMap Infiltrator`
- **Bio-Breach**: Malware `BioMetric Falsifier`, Tool `MedSys Probe`
- **Transport Chaos**: Malware `Vehicle Chaos Engine`

Tổng thiệt hại lên kệ đếm ra đúng 9 đồ chơi.

```
9
```

### 7. Trích xuất Hash của Malware rễ

> It appears that the threat actor has always used the same malware in their campaigns. What is its SHA-256 hash?

Tạt vô tab Links thuộc diện mạo gã Malware *NeuroStorm*, móc mỉa ra lủng lẳng một Indicator núp lùm kết dính khăng khít.

![NeuroStorm Indicator](/assets/images/htb/the-card/c73df9c5-16c1-49ca-829d-b11f9f0446d6.png)

Đục nhát click vào cái indicator này, ngó sang phần Details bóc trần bộ mặt thật của nó, và đập vào mặt là cái đoạn mã băm chát chúa.

![Indicator Details](/assets/images/htb/the-card/16bca68f-2957-413d-9141-fb689efaef8e.png)

```
7477c4f5e6d7c8b9a0f1e2d3c4b5a6f7e8d9c0b1a2f3e4d5c6b7a8f9e0d17477
```

### 8. Thọc tay C2 Server từ Hash

> Browse to the second IP:port address and use the CogWork Security Platform to look for the hash and locate the IP address to which the malware connects. (Credentials: nvale/CogworkBurning!)

Giã thẳng cổng access vào *CogWork Security Platform* như bài mớm sẵn.

![CogWork Security Platform](/assets/images/htb/the-card/5baaaec4-3718-4aaf-8c9f-c8a83bf4383c.png)

Quét thông tin quả Hash thúi lụm được từ câu 7 trên platform, cuộn thanh kéo xuống phần Communications là túm được phần rốn IP mà cục nợ malware này bắn dữ liệu trỏ tới. Rành rành đây là C2 Server chứ đâu ra.

![Malware C2 Server](/assets/images/htb/the-card/3ba0940f-1eb0-4781-938e-a1f581df8685.png)

```
74.77.74.77
```

### 9. Xới đường hầm làm Persistence

> What is the full path of the file that the malware created to ensure its persistence on systems?

Đảo lại tab Details rành rọt của malware từ nãy giờ, bới tận đáy cọng bùn lên vội xem mớ File Operations. Mòi thủ đoạn ăn mòn, ngâm dấm hệ thống lộ lộ.

![Persistence Logs](/assets/images/htb/the-card/6252a01d-0e95-4a39-b4dc-986a3019e525.png)

```
/opt/lilnunc/implant/4a4d_persistence.sh
```

### 10. Khoét sạch cơ sở hạ tầng lũ Attacker (TAs)

> Finally, browse to the third IP:port address and use the CogNet Scanner Platform to discover additional details about the TA's infrastructure. How many open ports does the server have?

Găm cái IP đục được vào nhào lên cỗ máy CogNet Scanner của lab, rọi cái scanner dằn mặt soi ra lố 11 cái port tét bẹt.

![CogNet Scanner](/assets/images/htb/the-card/a6b0959e-41c3-4c08-934d-91578e4d11fd.png)

```
11
```

### 11. Moi thông tin Organization

> Which organization does the previously identified IP belong to?

Quay vả vào mảng phân tích IP Details, phơi nhãn Organization thuộc về:

![IP Organization](/assets/images/htb/the-card/dd2dad21-4b19-48bc-a630-1c7b64f6e6c6.png)

```
SenseShield MSP
```

### 12. Bắt gặp Message sặc mùi cợt nhả (Cryptic Message)

> One of the exposed services displays a banner containing a cryptic message. What is it?

Thọt vào chỗ lách lòi service moi thông tin banner (Header phô mặt). Rớt lại nguyên một câu sặc mùi cà chớn mập mờ "cryptic", ịn chễm chệ ngay nòng port 7477 với quả service ngầm `unknown`.

![Cryptic Banner](/assets/images/htb/the-card/db28787c-beac-4fe6-b188-aba3d46338ff.png)

```
He's a ghost I carry, not to haunt me, but to hold me together - NULLINC REVENGE
```
